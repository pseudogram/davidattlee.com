What is a Web Server?

David Attlee

05 Dec 2018 • 2 min read

Simply, a web server processes incoming network requests. These requests may be one of a variety of different protocols UDP, TCP and HTTP (being the most common). Not to mention they can also act the first line of defence, securing connections, monitoring traffic and more. What’s more, once a request has been handled it responds by serving the appropriate content.

Break it down to Basics

Web Server, may refer to hardware, software, or the conjunction of the two. The hardware stores the web server software, a websites component files (HTML, CSS, JavaScript files, etc). And is connected to the internet. The software controls how web users access hosted files. By using at least an HTTP server. An HTTP server is software that understands URLs, and HTTP.
A web server can be accessed through domain names (like davidattlee.com). So, when a browser requests a file (hosted on a server), the browser requests a file via HTTP, using the domain name. The request reaches the correct web server (the hardware) and the HTTP server (software) accepts the request. After which it finds the requested file, and sends it back to the browser (via HTTP once again). If the file is unfound then a 404 response is returned.

Now theres two types of web servers, static and dynamic. We call one a static web server, or stack, when it sends files “as-is” to your browser (think craigslist). Alternatively, a dynamic web server uses extra software (eg. an application server and a database). We say “dynamic” because the server updates the hosted files before sending them. Most modern websites are like this as it makes it easier to set up, modify and maintain.

Give me an analogy. I don't get it.

Think of a web server like a hosts at a fancy restaurant. You arrive, they greet you, check your booking and guide you to your table. Once seated they bring you a menu, after which they handle all communications with the kitchen and bringing you each course. As you ask for it, When you ask for it. Occasionally, you may request something for the menu and the kitchen has run out (and that's your 404).

Ok, so how do I make one?

Dear god, don't make one yourself. There are plenty of great, free open source solutions out there. Take your pick. NGINX is currently the most popular, and incredibly easy to set up. However, if you're setting up a personal website and you hate yourself, try IIS.
